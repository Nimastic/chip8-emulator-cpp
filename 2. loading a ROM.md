before we can execute instructions we need to have those instructions in memory, 
so we need a function that loads contents of a ROM file

```cpp
#include <fstream>
/*
is used to read and write data to external files. It provides three primary stream classes to handle these operations: std::ifstream for reading, std::ofstream for writing, and std::fstream for doing both
*/

const unsigned int START_ADDRESS = 0x200;

void Chip8::LoadROM(char const* filename)
{
	// Open the file as a stream of binary and move the file pointer to the end
	std::ifstream file(filename, std::ios::binary | std::ios::ate);

	if (file.is_open())
	{
		// Get size of file and allocate a buffer to hold the contents
		std::streampos size = file.tellg();
		char* buffer = new char[size];

		// Go back to the beginning of the file and fill the buffer
		file.seekg(0, std::ios::beg);
		file.read(buffer, size);
		file.close();

		// Load the ROM contents into the Chip8's memory, starting at 0x200
		for (long i = 0; i < size; ++i)
		{
			memory[START_ADDRESS + i] = buffer[i];
		}

		// Free the buffer
		delete[] buffer;
	}
}

```

Chip8’s memory from 0x000 to 0x1FF is reserved, so the ROM instructions must start at 0x200
set the PC to 0x200 in the constructor because that will be the first instruction executed.
