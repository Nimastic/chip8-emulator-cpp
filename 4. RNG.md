There is an instruction which places a random number into a register

With physical hardware, this can be done by reading the whole value from a noisy disconnected pin or using a dedicated RNG chip, we'll use C++'s RNG

We’ll need to add two member variables and seed the RNG in the constructor. A simple seed is the system clock.

```cpp

#include <chrono>
#include <random>

[...]

class Chip8
{
	Chip8()
		: randGen(std::chrono::system_clock::now().time_since_epoch().count())
        /*
            this is a constructor member initialiser list
            initialises members as object is being built before {} body runs
            Order of event:
            1. Allocate Object
            2. Run initialiser (ranGen gets seeded)
            3. Run the constructor body {}
            Why not assign in the body? See below

        */
	{
		[...]

		// Initialize RNG
		randByte = std::uniform_int_distribution<uint8_t>(0, 255U);
	}

	[...]

	std::default_random_engine randGen;
	std::uniform_int_distribution<uint8_t> randByte;

    ```

We can then get a random number between 0 and 255 by using randByte.


Why not assign constructor member initialiser list in the body?
You can often do:

Chip8() {
    randGen = std::default_random_engine(seed);  // assignment after default construction
}
But the : member(value) form constructs it correctly the first time. That’s the normal way to seed engines, bind references, init const members, etc.
